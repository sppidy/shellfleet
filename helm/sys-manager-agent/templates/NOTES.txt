After approval, the agent appears in the dashboard's Kubernetes tab
on the agent named `{{ include "sys-manager-agent.fullname" . }}-id`.

⚠ The pairing token is stored at /etc/sys-manager/agent-token INSIDE
the running container. A Pod restart wipes it. Once you've paired,
copy the token into a Kubernetes Secret and re-install with
`token.existingSecret` set so the binding survives restarts:

  kubectl -n {{ .Release.Namespace }} exec deploy/{{ include "sys-manager-agent.fullname" . }} \
    -- cat /etc/sys-manager/agent-token > /tmp/agent-token
  kubectl -n {{ .Release.Namespace }} create secret generic {{ include "sys-manager-agent.fullname" . }}-token \
    --from-file=agent-token=/tmp/agent-token
  helm upgrade {{ .Release.Name }} sys-manager-agent \
    --set token.existingSecret={{ include "sys-manager-agent.fullname" . }}-token
